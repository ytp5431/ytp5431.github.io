HEXO博客搭建过程记录

Created2023-11-11|Updated2023-11-19
|Post Views:

前言：
本人非程序员活着电脑相关专业，只是单纯想记录一些生活中的一些感想或者杂项或者其他技术方面的事情，防止自己忘记。故想自建一个个人博客作为自己在互联网上的一个小据点，最好隐私性强一点。最开始想用QQ空间，但并不想让家人朋友看到这些，所以放弃了QQ空间。之后学习自己在互联网上搭建博客，以此作为记录：
开始选择博客搭建的工具，一开始是想着简单的方式最好像QQ空间一样，只要申请一下就可以，类似的只有CSDN之类的，但查了一下相关的资料CSDN之类的官方网站发表还要审核，不符合我的隐私要求，故放弃，以下为查到的博客搭建工具的信息，以后可能想玩的时候会再搞一搞。
CSDN——发表要审核。
typecho——看教程要申请器，最便宜的要一年八十多，而且好像挺复杂的，除了服务器还要买域名，还是要钱，想象算了。
wordpress——听说也要服务器和域名，但自己试了一下似乎不需要直接申请就行，但网页卡成PPT，也许是墙的原因根本加载不动。
HEXO——复杂对于非计算机专业的来说是真复杂，而且是静态的（我也不知道静态和动态有啥区别），但不要钱，故最终选择用HEXO。

HEXO搭建过程

注：所有命令可以查看https://hexo.io/zh-cn/获取详细信息
博客主题也可以在https://hexo.io/zh-cn/中查找

1. 需要的环境和程序安装

1.1 node.js安装

我也不知道node.js到底是个啥，只要装上就好，安装过程就是一直下一步，我选择的Windows版本：下载地址：http://nodejs.cn/download/
安装完成后查看是否有系统的环境变量，如果没有还要配置上系统的环境变量。
去命令提示符（WIN+R-CMD）中检查是否安装成功：
node -v
npm -v
如果出现版本号就是已经成功了。

1.1 Git安装

下载地址：https://git-scm.com/同样一直下一步直到安装完成。
安装完成后在桌面鼠标右键会看到’GIT GUI HERE’和’GIT BASH HERE’两个图标表示安装完成，点’GIT BASH HERE’进入一个类似命令提示符的图框内，就是Git的操作界面，输入git version，如果出现版本号就是安装成功。

1.1 HEXO安装

首先新建一个空的文件夹作为之后博客的本地文件夹，在空文件夹内右键然后点’GIT BASH HERE’进入命令提示符空间内，输入命令：
npm install -g hexo-cli
等一会，安装完成。
之后再在’GIT BASH HERE’输入初始化的命令（之后无特殊说明命令符就是在’GIT BASH HERE’中输入）：
hexo init
然后可以看到你新建的空文件夹中有了一些东西，这些就是博客产生的配置文件。

2. GitHub

GitHub是用来作为博客托管的网站，将博客网页建好后上传到GitHub上，并使用GitHub提供的域名进行访问。
也可以使用国内的GITEE等网站，由于GITEE为国内网站，访问过程中没有遇到卡顿或者加载不上的问题，但在申请上传的时候需要上传身份证照片及本人手持身份证照片，太麻烦了。

2.1 GitHub加速

GitHub由于在国外，访问经常出问题，需要一些方法进行加速，具体方法可以使用修改HOST或者使用加速器等

2.2 GitHub注册账号

访问https://github.com/ ，并申请一个账号，申请过程中需要填写邮箱等信息。

2.3 GitHub建立仓库

在自己的主页中的右上角点击＋下的New repository 建立一个新的仓库，注意仓库名称为username.github.io 的仓库(username 为你的 Github 用户名)，此处不能随意命名。Description随意
选择Public，分支branches建议选择单独分支，不然之后配置_config.yml文件时可能出错。完成后要等一会再刷新才能出现你的仓库，可能还是网络的问题。

2.4 增加密钥绑定账号

配置SSH密钥，同时在本地计算机和GitHub上配置以便上传博客信息，防止使用用户名和密码访问。
到桌面右键选择’GIT BASH HERE’打开命令输入界面，并输入命令：ssh-keygen -t rsa -C “你的github登录邮箱” 生成.ssh秘钥，之后会有三次提示，并按三次回车。注意密钥的邮箱要与github的注册邮箱相同，如果不同可以重新添加github邮箱。
生成密钥后去 C:\Users\您的用户名\ .ssh文件夹下找到.ssh\id_rsa.pub 文件，用TXT打开该文件，里面的内容就是你的共同密钥，复制里面的内容。
打开自己的github 主页，进入主页的右上角个人设置 -> SSH and GPG keys -> New SSH key（不是进入仓库的设置界面），把复制的内容粘贴进去，title 随便填，没有什么意义，保存即可，我们的公钥就添加成功了。注意：
输入命令: ssh -T git@github.com
如果没有设置成功（ssh: connect to host github.com port 22: Connection refused
fatal: Could not read from remote repository.），可能是端口22被占用，可以在 C:\Users\您的用户名\ .ssh文件夹下找到.ssh\下新建个文件’known_hosts’用txt打开并输入：
（github.com,20.205.243.166 ecdsa-sha2-nistp256 AAAAE2VjZHNhLXNoYTItbmlzdHAyNTYAAAAIbmlzdHAyNTYAAABBBEmKSENjQEezOmxkZMy7opKgwFB9nkt5YRrYMjNuG5N87uRgg6CLrbo5wAdT/y6v0mKV0U2w0WZ2YB/++Tpockg=[ssh.github.com]:443,[20.205.243.160]:443 ecdsa-sha2-nistp256 AAAAE2VjZHNhLXNoYTItbmlzdHAyNTYAAAAIbmlzdHAyNTYAAABBBEmKSENjQEezOmxkZMy7opKgwFB9nkt5YRrYMjNuG5N87uRgg6CLrbo5wAdT/y6v0mKV0U2w0WZ2YB/++Tpockg=）
并保存。然后重新输入命令: ssh -T git@github.com可以发现设置成功。

2.5 配置邮箱和用户名

输入命令git config –global user.name “ Github username”
输入命令: git config –global user.email “github 注册邮箱”

3. 博客的部署

3.1本地部署

输入命令：hexo init 进行初始化
输入命令：hexo c删除旧的HTML文件
输入命令: hexo g 会在 public 文件夹下生成相关的 html 文件，这些文件将来需要提交到 Github 上
输入命令：hexo s 可以开启本地预览服务，打开浏览器访问 http://localhost:4000 即可看到博客内容

3.2github上部署博客

到博客文件夹下的GIT输入命令: npm install hexo-deployer-git –save
编辑 D:\hexo 目录下的 _config.yml 文件，在文件末尾修改如下内容:
deploy:
type: git
repo: git@github.com:username/username.github.io.git//此处为仓库的 github 个人主页链接地址
branch: main//此处的内容与github的分支相同，否则无法部署成功
找到并修改url: https://username.github.io
在 D:\hexo 目录下，
输入命令: hexo g -d （也可以分开输入hexo g ；hexo d）将本地 blog 推送到 github 远程仓库，刷新主页，可能需要等几分钟刷新才能成功。

4.修改主题

部署完成后使用的主题为默认的主题，并不好看，可以到https://hexo.io/zh-cn/中选择自己喜欢的主题（注意选择最近还在维护的主题）
可以使用命令git clone 主题http链接 themes/主题名称 将主题下载至 themes 文件夹下，受制于网络问题，可能半天下载不下来，也可以去主题的github主页直接下载。下载下来的文件见复制粘贴到本地hexo下的themes文件夹下。
编辑 D:\hexo 目录下的 _config.yml 文件找到theme: landscape改为theme:“主题名”
根据主题说明修改H:\hexo\themes\“主题名”_config.yml下的相关主题配置。

5.写博客

hexo使用的.md文件有固定的格式和打开工具，需要使用VScode、notepad、markdown、等编辑软件打开编辑。
在D:\hexo\source文件夹下有两个文件夹（img、__post）如果没有自己新建生成：其中img文件夹下存放图片格式，_post文件夹下存放文章。
新建文章：在 D:\hexo 目录下，通过输入命令: hexo new “文章 title” 会在 /source 文件夹下生成对应文章的 .md 文件
